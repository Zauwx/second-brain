<objective>
`POST /ai/suggest-tags` returns HTTP 200 with `{"tags": []}` for every note against the real stack.
Root cause: `json_mode=True` maps to `format="json"`, under which llama3.2:3b emits an object whose
keys are copied verbatim from the prompt's placeholder example (`{"tag-one": "...", "tag-two": "..."}`).
`_parse_tag_list` only unwraps a dict when one of its values is a list — every value here is a string,
so it falls through and returns `[]`. Silently.

Fix: replace the `json_mode: bool` provider seam with a pass-through `format: str | dict = ""`
parameter, and have the AI domain pass an explicit JSON schema. That yields
`{"tags": ["rag", "retrieval", ...]}` — a shape the EXISTING parser already unwraps correctly
(3/3 clean runs verified live during the Phase 05 checkpoint).

Purpose: unblocks Phase 05 sign-off (STATE.md records this as BLOCKING).
Output: corrected provider seam, schema-constrained tag generation, observability on parse failure,
and an opt-in live regression test that closes the structural blind spot which let this reach a
live checkpoint through a 128-green suite.
</objective>
