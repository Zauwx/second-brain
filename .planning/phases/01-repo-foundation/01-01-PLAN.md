<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| working tree -> git history (PUBLIC repo) | Once committed and pushed to a public repo, secrets are irreversibly exposed in history |
| repo files -> Linux container runtime | Windows-authored text files cross into a Linux container; CRLF breaks shell interpreters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `.env` secrets reaching git history | mitigate | `.env` added to `.gitignore` in Task 1 BEFORE any commit exists; only `.env.example` with placeholders is committed (Task 2); verified downstream by `git log --all -- .env` returning nothing (Plan 03) |
| T-01-02 | Tampering / Denial | CRLF line endings corrupting shell scripts in Linux containers | mitigate | `.gitattributes` with `* text=auto eol=lf` + explicit `*.sh eol=lf` created in Task 1 |
| T-01-03 | Information Disclosure | Real secret accidentally pasted into committed `.env.example` | mitigate | Acceptance criterion forbids real key strings; all values are obvious placeholders (Task 2) |
</threat_model>
