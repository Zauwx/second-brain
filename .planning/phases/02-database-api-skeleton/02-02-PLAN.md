## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | Untrusted JSON request bodies and path/query params cross into the notes router (no auth this phase — open endpoints, intentional, Phase 3 adds auth) |
| API → MySQL | Service/repository issue queries against MySQL on behalf of any caller |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering (SQL injection) | repository queries, `?filter=` LIKE | mitigate | all queries use SQLAlchemy expression API / parameter binding (`select(Note).where(...)`, `Note.content.ilike(f"%{filter}%")` binds the value as a parameter, not string-concatenated SQL) |
| T-02-07 | Denial of Service | POST/PUT request body size, `GET /notes/?size=` | mitigate | `size` bounded `ge=1, le=100` via `Query` → over-max returns 422 (D-05); page `ge=1`. Body size: rely on default ASGI/uvicorn limits (note in SUMMARY for Phase 7 review) |
| T-02-08 | Information Disclosure | error responses | mitigate | FastAPI default error bodies only (D-19) — `{"detail":"Note not found"}` for 404, standard 422 validation body; no stack traces, no DB internals leaked |
| T-02-09 | Repudiation | note mutations | accept | no audit log this phase; server-managed created_at/updated_at provide basic timestamps; full auditing out of scope (no auth yet) |
| T-02-10 | Elevation of Privilege | cross-note access | accept | endpoints are intentionally open this phase (no users yet); per-user isolation is Phase 3 (AUTH-04) — do NOT introduce auth here |
| T-02-SC | Tampering | uv install testcontainers[mysql] | mitigate | testcontainers is a well-known PyPI package in the project's locked dev stack (CONTEXT.md D-16); version pinned; not `[ASSUMED]`/`[SUS]` |
</threat_model>
