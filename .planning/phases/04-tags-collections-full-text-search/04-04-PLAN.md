| client → GET /search | Untrusted `q` flows into MATCH ... AGAINST ... IN BOOLEAN MODE |
| API → MySQL FULLTEXT | Parameterized match expression; user_id scoping enforced |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-inj | Tampering | search_fulltext MATCH AGAINST | mitigate | `match(against=q)` binds q as a parameter (no string interpolation); `sanitize_boolean_query` strips @/++/+- so malformed operators cannot cause syntax errors; verified by test_search_sanitization |
| T-04-iso | Information Disclosure | GET /search results | mitigate | `.where(Note.user_id == user_id)` on every search query; verified by test_search_isolation (total == 0 for other user) |
| T-04-08 | Denial of Service | operator abuse / huge q | mitigate | Sanitizer removes error-causing operator combos; pagination caps result size; q min_length=1 |
| T-04-09 | Spoofing | GET /search | mitigate | get_current_user JWT dependency (401) inherited from Phase 3 |
| T-04-SC | Tampering | dependency installs | accept | No new packages this phase |
</threat_model>
