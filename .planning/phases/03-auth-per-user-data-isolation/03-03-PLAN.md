Enforce strict per-user data isolation: protect every note endpoint with `get_current_user`, scope all note reads/writes to the authenticated owner, and add the `notes.user_id` FK. After this slice a user can only ever see, edit, or delete their own notes — proven by cross-user integration tests against real MySQL.

Purpose: Implements AUTH-04 (every query scoped to the authenticated user). This is the final isolation slice — it consumes get_current_user (Plan 02) and the User model (Plan 01), threads ownership through the notes domain, and migrates the notes table.
Output: notes.user_id FK migration; protected + owner-scoped notes endpoints (403 cross-user, 404 missing, 401 no-token); existing note tests upgraded to authenticated; new tests/test_notes_isolation.py proving isolation.
</objective>
