<objective>
Complete the token lifecycle so a user stays authenticated without re-entering credentials and can log out. Adds the `get_current_user` HTTPBearer dependency (the gate every protected route will use), `POST /auth/refresh` with rotation (revoke old jti, issue new pair), and `POST /auth/logout` (revoke jti → 204). Establishes correct 401 semantics for missing/expired/invalid tokens.

Purpose: Implements AUTH-03 (token refresh + stay-authenticated). Produces `get_current_user`, which Plan 03 threads through the notes domain to enforce per-user isolation. Depends on Plan 01's AuthService/AuthRepository/User+RefreshToken models and the refresh_tokens table.
Output: A user can refresh a rotating token, log out, and is correctly rejected (401) with no/expired/invalid tokens. test_auth.py grows refresh/logout/401 coverage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → protected routes | Bearer access token crosses here; validated stateless then DB-checked for user existence |
| HTTP client → /auth/refresh, /auth/logout | Refresh token crosses here; signature + exp + DB jti revocation state checked |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Spoofing | Missing/forged Authorization header | mitigate | HTTPBearer(auto_error=False) + explicit None check → 401 (Pitfall 3); signature verified HS256 — Task 1 |
| T-03-09 | Elevation of Privilege | Expired access token replay | mitigate | jwt.decode enforces exp; ExpiredSignatureError → 401 — Task 1 |
| T-03-10 | Repudiation / Replay | Refresh-token replay after rotation | mitigate | jti persisted in refresh_tokens; rotation revokes old jti; replay of revoked/absent jti → 401 (D-04/D-06) — Task 2 |
| T-03-11 | Elevation of Privilege | Refresh token reused after logout | mitigate | logout flips jti.revoked; subsequent refresh → 401 (D-05) — Task 2 |
| T-03-12 | Elevation of Privilege | Stale token after user deletion | mitigate | get_current_user does a DB lookup by sub; missing user → 401 — Task 1 |
| T-03-13 | Information Disclosure | Mutable claims (email/role) embedded in JWT | mitigate | Access/refresh payloads carry only sub + exp (+ jti on refresh); user attrs fetched from DB — Tasks 1-2 (Pitfall 4) |
| T-03-14 | Elevation of Privilege | Cross-login token revocation collateral | accept | D-03: refreshing one token must NOT revoke siblings; cascade/family reuse-detection deferred (D-06) — verified by test_other_refresh_tokens_unaffected_by_rotation |
</threat_model>
