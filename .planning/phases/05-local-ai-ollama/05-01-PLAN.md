## Phase Goal

**As a** self-hosting note-taker, **I want to** run a local LLM (Ollama) as a private Docker service and trigger automatic summarization and tag suggestion on my own notes, **so that** I get AI help on my knowledge base with zero cloud calls, zero billing, and provable privacy.

_(Derived from ROADMAP Phase 5 goal; the phase decomposes into a foundation enabler — this plan + 05-02 — and two feature slices, 05-03 summarize and 05-04 suggest-tags.)_

<objective>
Stand up the Ollama Docker service and install the two new Python dependencies that the summarize and suggest-tags slices both depend on. This is the infra half of the foundation slice.

Purpose: Both AI features (05-03, 05-04) call a local LLM running in a sibling container; this plan makes that container exist (memory-bounded, internal-network-only) and makes the `ollama` + `tenacity` client libraries available.
Output: A running `ollama` Docker service, the two new deps in the lockfile, and tunable Ollama settings + documented env keys.
</objective>
