## Phase Goal

**As a** self-hosting note-taker, **I want to** run a local LLM (Ollama) as a private Docker service and trigger automatic summarization and tag suggestion on my own notes, **so that** I get AI help on my knowledge base with zero cloud calls, zero billing, and provable privacy.

<objective>
Build the provider abstraction seam that both AI features call through, expose Ollama reachability on `/health` (criterion 1), and stand up the Wave 0 test infrastructure (a hand-rolled `FakeLLMProvider` injected via dependency override) so the summarize and suggest-tags slices can be tested with zero real Ollama calls.

Purpose: This is the seam half of the foundation slice. It delivers a real user-observable capability now — `GET /health` tells you whether local AI is available — while defining the `LLMProvider` contract that 05-03/05-04 implement against and Phase 6's `AnthropicProvider` will slot into unchanged.
Output: `app/ai/providers/` (protocol + OllamaProvider), `get_llm_provider` DI seam, extended `/health`, and the `FakeLLMProvider`/`ai_client` test fixtures.
</objective>
