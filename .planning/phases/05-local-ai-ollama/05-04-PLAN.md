## Phase Goal

**As a** self-hosting note-taker, **I want to** run a local LLM (Ollama) as a private Docker service and trigger automatic summarization and tag suggestion on my own notes, **so that** I get AI help on my knowledge base with zero cloud calls, zero billing, and provable privacy.

<objective>
Deliver the suggest-tags feature slice end-to-end: `POST /ai/suggest-tags` resolves the caller's own note (404/403), asks the local LLM for a JSON list of tags, leniently parses the (possibly messy) output into `list[str]`, and returns it — suggest-only, no attach, no persist (D-04). Then verify the whole phase live.

Purpose: Second headline capability of Phase 5 (AIL-02, criterion 3). The user attaches any tags they like via the existing Phase-4 `POST /notes/{id}/tags` endpoint — the human stays in the loop. The closing checkpoint proves all five ROADMAP criteria hold against a real running stack.
Output: `AIService.suggest_tags` + `_parse_tag_list`, `POST /ai/suggest-tags`, its test suite, and a signed-off live demo.
</objective>
