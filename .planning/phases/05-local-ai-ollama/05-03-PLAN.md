## Phase Goal

**As a** self-hosting note-taker, **I want to** run a local LLM (Ollama) as a private Docker service and trigger automatic summarization and tag suggestion on my own notes, **so that** I get AI help on my knowledge base with zero cloud calls, zero billing, and provable privacy.

<objective>
Deliver the summarize feature slice end-to-end: `POST /ai/summarize` resolves the caller's own note (404/403), sends its content to the local LLM through the provider seam, persists the returned 2-3 sentence summary on a new `Note.summary` column, and surfaces it on `GET /notes/{id}`.

Purpose: This is the first headline user capability of Phase 5 (AIL-01, criterion 2) — a real user can now generate and store an AI summary of their note. It exercises another Alembic migration (a project learning goal) and establishes the `AIService` that 05-04 extends.
Output: Migration 0006 + `Note.summary`, `AIService.summarize`, `POST /ai/summarize`, and its test suite.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /ai/summarize | JWT-authenticated; must operate only on the caller's own note |
| note content → LLM prompt | Stored note text becomes prompt input (first-party, but shape the habit) |
| api → ollama | Provider call may fail/timeout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Elevation of Privilege | POST /ai/summarize note resolution | mitigate | Reuse `NoteService.get_or_404_owned` (403/404) — no separate ownership check (Task 3, ASVS V4) |
| T-05-04 | Tampering | note content injected into the LLM prompt | mitigate | Wrap content in explicit prompt delimiters + defensively `.strip()` output; model is local+offline so it cannot exfiltrate (Task 3) |
| T-05-02 | Denial of Service | large note content / long inference | mitigate | Truncate content to a bounded max before prompting; bounded tenacity retry + finite timeout cap per-request cost (single-user scale — no rate limiter, consciously accepted) |
| T-05-05 | Denial of Service | Ollama down/erroring | mitigate | `_safe_complete` catches connection/timeout/ResponseError → clean 503; note CRUD unaffected (Task 3, D-07) |
</threat_model>
